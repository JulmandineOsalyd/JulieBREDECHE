The question comes up often: **can you create Copilot agents without a license?**

The answer is yes, but with very different capabilities depending on the situation. Here are the 3 scenarios I encounter with my clients, and what they actually allow.

---

## Scenario 1: No Copilot License

Both creator and users only have the **free Copilot Chat** included with Microsoft 365.

**What the agent can do:**
- Answer from public websites
- Access online documentation available without authentication

⚠️&nbsp; **Major limitation:** no access to your Microsoft tenant data (SharePoint, OneDrive, Teams, etc.)

✅&nbsp; **Ideal for:** industry monitoring agents, FAQs based on public documentation.

---

## Scenario 2: Creator Has a Copilot M365 License

The creator has **Copilot for Microsoft 365** — users keep the free Copilot Chat.

**What the agent can do:**
- Access public websites
- Answer from **files provided directly** to the agent (Word, PDF, PowerPoint)

**Key point:** users can use the agent **without a personal paid license**.

⚠️&nbsp; **Major limitation:** documentation doesn't update automatically. Files loaded directly are accessible to all agent users, regardless of their SharePoint permissions.

✅&nbsp; **Ideal for:** stable internal procedures, reference guides that rarely change.

> This is the most common scenario in my client portfolio. Organizations are often in a pilot phase: testing capabilities and measuring value before investing in licenses at a larger scale.

---

## Scenario 3: Copilot for Microsoft 365 at Scale

Both creator and users have **Copilot for Microsoft 365** licenses.

**What the agent can do:**
- Access the entire corporate cloud: SharePoint, OneDrive, Teams, Dataverse…
- Strictly respect existing access permissions

⭐️&nbsp; **Key point:** each user only sees what they have access to. Document security is preserved.

✅&nbsp; **Ideal for:** Level 1 IT support, new employee onboarding, large-scale HR FAQs.

---

## Summary

| | No Copilot License | Creator with Copilot M365, users with Copilot Chat | Copilot for M365 for Everyone |
|---|---|---|---|
| **Accessible sources** | Public websites, online documentation | Public websites + files provided to the agent | Entire corporate cloud (SharePoint, OneDrive, Teams) |
| **Access rights** | Not applicable | Users access loaded files regardless of their SharePoint rights | Existing SharePoint rights strictly respected |
| **Typical use cases** | Monitoring, FAQ on public documentation | Stable internal procedures, reference guides | Level 1 IT support, onboarding, HR FAQs |

## And What About Copilot Studio?

Declarative agents often represent the **first step** before broader Copilot license adoption and the integration of advanced agents via Copilot Studio.

Copilot Studio enables going even further: creating agents that **trigger actions**, orchestrate workflows, and interact with business systems.

We move from simple information retrieval to **action execution**. That's the boundary between declarative agents and advanced agents.
