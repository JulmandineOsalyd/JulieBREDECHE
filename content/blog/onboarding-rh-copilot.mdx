Il y a deux ans, j'accompagnais la migration d'un cabinet de conseil vers OneDrive. 4 000 utilisateurs, 2 To de données, 18 mois de projet. Le lancement s'est bien passé — jusqu'à ce que le support informatique commence à recevoir des dizaines de tickets par jour avec le même symptôme : des fichiers qui "disparaissaient" ou refusaient de se synchroniser.

Après investigation, la cause était systématiquement la même : des chemins de fichiers trop longs.

## Le problème des chemins trop longs dans OneDrive

Windows et OneDrive ont une limitation historique : le chemin complet d'un fichier (dossier racine + tous les sous-dossiers + nom du fichier + extension) ne peut pas dépasser **260 caractères** pour la plupart des opérations.

Cette limite n'est pas un problème sur un serveur de fichiers classique où les utilisateurs travaillent directement. Mais quand vous synchronisez ce serveur vers OneDrive, le chemin local du client OneDrive s'ajoute au chemin d'origine. Résultat : des fichiers qui fonctionnaient parfaitement sur le serveur dépassent soudainement la limite.

Exemple concret :

```
Serveur d'origine :
\\SERVEUR\Partages\Direction Générale\Projets 2023\Appels d'offres\Lot 3 - Travaux de rénovation\Documents finaux\v3\Offre_technique_finale_corrigée_validée_DG.docx
→ 142 caractères ✅

Même fichier dans OneDrive :
C:\Users\jean.dupont\OneDrive - Mon Entreprise\Direction Générale\Projets 2023\...
→ 198 caractères ✅ mais proche de la limite

Avec un sous-dossier supplémentaire habituel :
C:\Users\jean.dupont\OneDrive - Mon Entreprise SAS\Direction Générale - Siège\...
→ 273 caractères ❌ ÉCHEC DE SYNCHRONISATION
```

Le pire : OneDrive ne notifie pas toujours clairement l'utilisateur. Le fichier reste sur le serveur, ne se synchronise pas, et l'utilisateur ne le retrouve pas sur son autre poste. De là vient la sensation que les fichiers "disparaissent".

## Pourquoi aucun outil natif ne suffit

Microsoft fournit des rapports dans le centre d'administration OneDrive, mais ils n'affichent les erreurs qu'*après* la tentative de synchronisation. Pour une migration, c'est trop tard.

J'ai cherché des outils de diagnostic préventif. Les solutions existantes étaient soit payantes, soit nécessitaient des permissions administrateur, soit produisaient des rapports illisibles.

J'ai donc créé **OneDrive Path Checker** : un outil gratuit qui tourne directement dans votre navigateur.

## Comment fonctionne OneDrive Path Checker

L'outil utilise l'**API File System Access** du navigateur (disponible dans Chrome, Edge et les navigateurs Chromium récents). Cette API permet à une page web de lire la structure d'un dossier local avec la permission explicite de l'utilisateur — sans envoyer aucun fichier sur Internet.

Le fonctionnement est simple :

1. Vous cliquez sur "Choisir un dossier" et sélectionnez votre dossier source (le dossier que vous prévoyez de migrer vers OneDrive)
2. L'outil parcourt récursivement toute l'arborescence
3. Pour chaque fichier, il calcule la longueur du chemin **tel qu'il sera dans OneDrive** (en ajoutant le préfixe OneDrive simulé)
4. Il liste tous les fichiers dont le chemin dépasse le seuil configuré
5. Vous exportez la liste en CSV pour planifier le nettoyage

**Aucun fichier n'est lu ni envoyé** — seuls les noms et chemins sont analysés.

## Ce que j'ai appris en l'utilisant sur des vraies migrations

Depuis que j'utilise cet outil systématiquement avant chaque migration, j'ai collecté quelques statistiques intéressantes :

- En moyenne, **12% des fichiers** ont un chemin qui dépasse 220 caractères dans les arborescences de plus de 5 ans
- Les pires contrevenants sont presque toujours les mêmes dossiers : "Appels d'offres", "Archives", "Documents juridiques"
- Les noms de fichiers contenant des versions ("_v2_finale_corrigée_relue") sont responsables de 60% des dépassements
- Les organisations qui ont migré sans diagnostic préalable ont en moyenne **3x plus de tickets support** dans les 30 premiers jours

## Bonnes pratiques pour éviter le problème à la source

Même avec l'outil, le vrai travail est de traiter les fichiers problématiques. Voici ce que je recommande :

**Avant la migration :**
- Exécutez OneDrive Path Checker sur tout le périmètre de migration
- Triez le rapport CSV par longueur de chemin décroissante
- Traitez d'abord les dossiers les plus profonds (renommez ou aplatissez)

**Conventions de nommage à adopter :**
- Pas de numéros de version dans le nom de fichier → utilisez l'historique des versions SharePoint
- Longueur maximale recommandée pour un nom de fichier : 50 caractères
- Évitez les caractères spéciaux : `# % & * : < > ? / \ { | }`
- Préférez les tirets aux espaces dans les noms de dossiers

**Configuration OneDrive à vérifier :**
- Activez la limite de chemin étendue (LongPathsEnabled) dans la stratégie de groupe si possible
- Choisissez un nom de tenant court (le nom après "OneDrive - " compte dans le chemin)

L'outil est accessible gratuitement sur [la page dédiée](/outils/onedrive-path-checker). Aucune installation, aucun compte requis.
