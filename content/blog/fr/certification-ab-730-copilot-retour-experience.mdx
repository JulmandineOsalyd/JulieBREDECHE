J'ai passé la certification Microsoft AB-730. Ce que cette certification teste, ce n'est pas la capacité à écrire de bons prompts. C'est la compréhension de ce qui se passe *autour* : comment Copilot accède aux données, comment les droits d'accès et les labels de sensibilité conditionnent ses réponses, comment il s'articule entre les apps Microsoft 365.

---

## C'est quoi l'AB-730 ?

L'AB-730 s'appelle officiellement **Microsoft Certified : AI Business Professional**.

C'est une certification pensée pour les professionnels qui accompagnent le déploiement de Copilot dans les organisations : consultants M365, responsables adoption, change managers, chefs de projet. Pas pour les admins techniques ou les développeurs.

> Le détail de l'examen est disponible sur la [page officielle Microsoft Learn](https://learn.microsoft.com/fr-fr/credentials/certifications/resources/study-guides/ab-730).

---

## Au programme : gouvernance, agents et contenu

### 1. Fondamentaux de l'IA générative (25-30%)

L'examen teste la compréhension de **comment Copilot accède aux données** avec un focus sur les :

- **Droits d'accès aux fichiers** : Copilot ne voit que ce que l'utilisateur a le droit de voir. Si un fichier stocké dans SharePoint n'est pas accessible à l'utilisateur, Copilot ne l'indexe pas.
- **Labels de sensibilité** : Copilot respecte les labels de sensibilité appliqués aux fichiers de l'organisation.
- **Risques à connaître** : hallucinations, injection de prompt, dépendance excessive aux réponses IA.

### 2. Gérer les prompts et les conversations (35-40%)

L'examen couvre la gestion des prompts au quotidien : **référencer les bons fichiers, sauvegarder et planifier l'exécution d'un prompt, le partager avec un collègue.**

Mais la partie la plus intéressante, c'est les agents. L'examen teste la capacité à :

- Choisir entre un agent existant (ex: Researcher) et la création d'un agent sur mesure
- Configurer un agent : sources de connaissance, instructions, prompts suggérés
- Partager un agent avec les membres de son organisation

### 3. Générer et analyser du contenu (25-30%)

La dernière partie couvre quelque chose que j'observe souvent mal compris sur le terrain : **comment passer d'une app à une autre avec Copilot**.

- Générer un document dans Word depuis un prompt
- Déplacer des données et insights entre les apps Microsoft 365
- Utiliser Copilot Pages pour collaborer sur un contenu
- Exploiter les résumés de réunion Teams dans un flux de travail

---

## Ce que j'en retire en tant que consultante M365

Au delà de valider mes compétences existantes, le passage de cette certification m'a permis de définir **une grille de lecture plus formalisée**.

Quand un client me dit *"Copilot ne trouve pas nos fichiers"*, je sais maintenant structurer le diagnostic :

1. Les fichiers sont-ils accessibles à l'utilisateur (permissions SharePoint) ?
2. Ont-ils un label de sensibilité restrictif ?
3. Sont-ils dans le périmètre indexé par Microsoft 365 ?

Avant, j'aurais posé les mêmes questions par réflexe terrain. Maintenant, je les pose dans le bon ordre, avec le vocabulaire certifié pour l'expliquer au client.

> **La certification ne remplace pas l'expérience. Elle lui donne un cadre.**

---

## Ma conclusion

Trois constats après avoir validé cette certification :

1. **La gouvernance des données conditionne tout.** Avant de déployer Copilot, les droits d'accès et les labels de sensibilité doivent être bien gérés.
2. **L'orchestration inter-apps est sous-estimée.** La vraie valeur de Copilot, ce n'est pas dans une seule app. C'est dans la fluidité entre Outlook, Teams, Word, SharePoint.
3. **Cette certification crédibilise mes échanges avec les clients.** Pas comme diplôme affiché. Comme preuve qu'on a formalisé les bons sujets.
