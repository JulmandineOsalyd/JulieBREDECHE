C'est l'une des questions que l'on me pose le plus souvent lors de mes missions : *"On ne peut pas se permettre des licences Copilot M365 pour tout le monde — est-ce qu'on peut quand même utiliser Copilot Studio ?"*

La réponse est **oui**, et c'est plus simple qu'il n'y paraît. Voici ce qu'il faut savoir.

## Le modèle de licences Copilot Studio : deux scénarios distincts

Microsoft distingue deux modes d'utilisation de Copilot Studio :

**Scénario 1 : Agents dans l'écosystème M365 (Teams, Copilot)**
Pour déployer un agent directement dans Microsoft Teams ou dans l'interface Copilot M365, les utilisateurs finaux doivent effectivement avoir une licence **Copilot for Microsoft 365** (anciennement M365 Copilot). C'est ce scénario qui crée la confusion.

**Scénario 2 : Agents en dehors de l'écosystème M365**
Pour déployer un agent sur un site web, une application interne, via une API ou dans Teams *sans* passer par l'interface Copilot, les utilisateurs finaux **n'ont pas besoin de licence Copilot**. Seul le *créateur* de l'agent a besoin d'une licence Copilot Studio (environ 200€/mois pour une capacité de base).

C'est ce second scénario qui ouvre des possibilités considérables pour les organisations qui ne peuvent pas justifier le coût des licences Copilot pour l'ensemble de leurs collaborateurs.

## Ce que vous pouvez faire sans licence Copilot M365

Voici des cas d'usage concrets déployables avec uniquement une licence Copilot Studio :

### 1. Chatbot d'assistance interne sur SharePoint

Intégrez un agent directement dans votre intranet SharePoint Online via le composant WebPart dédié. Vos collaborateurs peuvent interroger la base de connaissances de l'entreprise, obtenir des réponses sur les procédures RH, les outils IT, les politiques internes — sans quitter SharePoint.

```
Canal de déploiement : SharePoint WebPart
Licence requise (utilisateurs) : Aucune
Licence requise (créateur) : Copilot Studio
Accès aux données : SharePoint, Dataverse, connecteurs Power Platform
```

### 2. Assistant client sur votre site web

Un agent conversationnel embarqué sur votre site web (code snippet HTML) pour qualifier les leads, répondre aux FAQ, prendre des rendez-vous via Power Automate. Aucun de vos prospects n'a besoin de licence Microsoft.

### 3. Agent dans Teams sans canal Copilot

Vous pouvez déployer un agent dans Teams comme une application (bot Teams) sans passer par le canal "Microsoft Copilot". Dans ce cas, les utilisateurs interagissent avec votre agent dans une fenêtre de chat dédiée — aucune licence Copilot M365 requise.

### 4. API REST pour intégration custom

Copilot Studio expose une API REST qui permet d'intégrer votre agent dans n'importe quelle application : portail client, application PowerApps, système de tickets, etc.

## Les limites à connaître

Soyons honnêtes : le mode sans licence Copilot M365 a des contraintes réelles.

**Pas d'accès aux données M365 sans configuration**
Sans licence Copilot, l'agent ne peut pas accéder nativement aux emails, calendriers ou fichiers OneDrive de l'utilisateur. Vous devrez configurer des connecteurs Power Platform explicites pour accéder à ces données.

**Modèle de tarification à la conversation**
En mode "Messages" (pay-as-you-go), chaque échange avec l'agent consomme des "messages" facturés. Le coût reste faible (environ 0,01€ par message selon le modèle), mais il faut l'anticiper dans votre budget.

**Fonctionnalités Copilot M365 non disponibles**
Les fonctionnalités qui requièrent le "contexte M365" (résumé de votre boîte mail du jour, analyse de vos réunions récentes…) sont réservées aux licences Copilot M365.

## Exemple concret : agent RH pour une PME de 150 personnes

Voici comment j'ai structuré un déploiement récent :

**Contexte** : PME industrielle, 150 salariés. Budget licences : 1 licence Copilot Studio (créateur). Zéro licence Copilot M365.

**Solution déployée** :
- Agent Copilot Studio connecté à une base de connaissances SharePoint (politiques RH, procédures, FAQ)
- Déployé via WebPart sur la page d'accueil de l'intranet SharePoint
- Connecté à Power Automate pour les demandes de congés simples (création automatique de l'entrée dans la liste SharePoint + notification au manager)

**Résultats** :
- 85% des questions RH de premier niveau traitées sans intervention humaine
- Coût mensuel : ~180€ (licence Copilot Studio) + ~15€ de messages consommés
- ROI atteint en moins de 2 mois vs. le temps RH économisé

## Checklist avant de démarrer

Avant de lancer votre premier agent sans licence Copilot M365, vérifiez ces points :

- [ ] Qui sera le créateur/administrateur de l'agent ? (1 licence Copilot Studio requise)
- [ ] Quel canal de déploiement ? (SharePoint, Teams bot, site web, API)
- [ ] Quelles sources de données ? (SharePoint OK nativement, autres via connecteurs)
- [ ] Estimation du volume de conversations par mois → calcul du coût messages
- [ ] Politique de sécurité : l'agent doit-il être accessible à tous ou authentifié ?
